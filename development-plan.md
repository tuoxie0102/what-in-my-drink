# 《记忆奶茶铺》开发计划

> 版本：v1.0  
> 日期：2026-06-20  
> 配套文档：product-architecture.md

---

## 一、总体路线

```
Phase 0：数据准备    →  整理数据文件，确认缺失项
Phase 1：核心闭环    →  首页→菜单→摇奶茶→掉落→卡片→结束
Phase 2：每日机制    →  每日3杯限制、收藏柜、进度持久化
Phase 3：店员系统    →  台词库、随机姿势、求助小游戏
Phase 4：视觉打磨    →  双轨风格精调、彩蛋主题、动效
Phase 5：测试发布    →  多设备测试、替换资产、部署
```

## 三、Phase 1：核心闭环（3-4天）

### 3.1 目标

跑通用户从"进入首页"到"看完卡片离开"的完整流程，**不做每日限制、不做收藏柜**。

### 3.2 页面实现顺序

```
Step 1: 首页（像素奶茶店 + 店员）
Step 2: 奶茶菜单（3杯选择）
Step 3: 摇奶茶动画
Step 4: 掉落物揭晓
Step 5: 记忆卡片（含现状）
Step 6: 店员吐槽 + 结束
```

### 3.3 技术选型

| 层级 | 方案 | 理由 |
|------|------|------|
| 渲染 | 纯 HTML/CSS/JS | 零依赖，用户可直接打开 |
| 像素风 | CSS `image-rendering: pixelated` + 像素字体 | 轻量，支持替换 |
| 奶茶店场景 | CSS Grid 布局 + 绝对定位像素元素 | 可替换背景图 |
| 卡片页面 | 独立 CSS 主题（千禧画报风） | 与像素风解耦 |
| 动画 | CSS animation + 少量 JS 触发 | 摇奶茶、掉落 |
| 数据加载 | `<script src="data/...">` 全局注入 | 简单，无需打包工具 |

### 3.4 文件结构

```
/
  index.html              ← 入口，加载所有 JS/CSS
  /css/
    pixel-theme.css       ← 像素奶茶店主题
    menu-theme.css        ← 菜单/WinXP风
    card-theme.css        ← 千禧画报风（记忆卡片）
    animations.css        ← 摇奶茶、掉落动画
    reset.css             ← 基础重置
  /js/
    app.js                ← 主控制器（页面切换逻辑）
    shop.js               ← 首页奶茶店渲染
    menu.js               ← 菜单选择交互
    shake.js              ← 摇奶茶逻辑
    card.js               ← 记忆卡片渲染
    clerk.js              ← 店员台词选择
    storage.js            ← localStorage 封装（Phase 2才用）
  /data/
    drinks.js             ← 奶茶数据
    cards.js              ← 卡片数据
    clerk-lines.js        ← 台词数据
    config.js             ← 配置
  /assets/
    /images/              ← 图片资产（见 product-architecture.md §7）
```

### 3.5 卡片页渲染规格

记忆卡片是情绪高潮，需特别处理：

- 千禧年画报风 CSS：柔和阴影、略微褪色感、暖黄基底
- 字体：标题用衬线体/手写体（模拟杂志），正文用无衬线
- 布局：卡片式，从上到下依次展示——
  1. 物品名称 + 年代
  2. 物品描述（可预留插图位）
  3. 背面内容（引用块，模拟手写/印刷）
  4. 时刻描述（小号正文，居中）
  5. 现实背景 + 现状（折叠/展开，小号灰色）
  6. 店员备注（底部，像素字体，保留奶茶店感）

### 3.6 验收标准

- [ ] 用户可看到像素奶茶店和店员
- [ ] 点击店员 → 出现 3 杯奶茶可选
- [ ] 选择后 → 摇奶茶动画 → 掉落物品
- [ ] 点击查看 → 千禧画报风卡片完整展示
- [ ] 店员台词显示 → 返回首页
- [ ] 同一会话可反复测试不同奶茶

## 四、Phase 2：每日机制（2天）

### 4.1 目标

加入每日限制、收藏柜、进度持久化。

### 4.2 具体任务

| # | 任务 | 说明 |
|---|------|------|
| 2.1 | `localStorage` 封装 | 读写收藏列表、今日日期、已喝杯数 |
| 2.2 | 每日重置逻辑 | 检测日期变化 → 重置 `daily_count` |
| 2.3 | 3杯限制 | 第3杯后触发"今天卖完了"+ 随机歇业理由 |
| 2.4 | 已收集去重 | 已获得的卡片不再出现（或在出现时替换） |
| 2.5 | 收藏柜页面 | 像素展示柜，按编号排列已收集卡片 |
| 2.6 | 收藏柜满 62 件彩蛋 | 店员特殊台词 |
| 2.7 | 特惠奶茶逻辑 | 3杯中固定1杯为特惠，触发二次确认弹窗 |

### 4.3 数据流

```
页面加载
  ↓
读取 localStorage → { collected, lastDate, dailyCount }
  ↓
if lastDate !== today:
    dailyCount = 0
    lastDate = today
  ↓
if dailyCount >= 3:
    显示 "今天卖完了"
else:
    显示可购买奶茶（从62张卡片中筛选未收集项，随机选3个分配）
  ↓
用户购买 → dailyCount += 1 → 存入 localStorage
  ↓
获卡后 → collected.push(cardId) → 存入 localStorage
```

### 4.4 验收标准

- [ ] 每天只能喝 3 杯
- [ ] 第二天自动重置
- [ ] 已收集卡片出现在收藏柜
- [ ] 已收集的不会再次掉落（或在掉落时有特殊处理）
- [ ] 特惠奶茶有二次确认弹窗

## 五、Phase 3：店员系统（2-3天）

### 5.1 目标

让店员"活"起来——随机姿势、丰富台词、求助小游戏。

### 5.2 具体任务

| # | 任务 | 说明 |
|---|------|------|
| 3.1 | 店员随机姿势 | 每次进入首页随机：煮珍珠/看漫画/趴着睡 |
| 3.2 | 店员随机台词 | 从 `clerk-lines.js` 按场景调取（开场/售卖中/售罄/空闲） |
| 3.3 | 暂停营业彩蛋 | 极小概率首页显示暂停营业理由 |
| 3.4 | 找吸管小游戏 | 桌面杂物中点击吸管 → 奖励卡片 |
| 3.5 | 帮忙封口小游戏 | 拖动封口膜 → 奖励卡片 |
| 3.6 | 整理柜台小游戏 | 拖动物品归位 → 奖励卡片 |
| 3.7 | 求助触发概率控制 | 10-20%，每日最多1次 |

### 5.3 验收标准

- [ ] 店员每次出现姿势不同
- [ ] 台词在不同场景切换自然
- [ ] 三种求助小游戏均可触发并正常走完流程
- [ ] 求助不占每日 3 杯额度

## 九、风险与建议

### 9.1 关键风险

| 风险 | 缓解措施 |
|------|----------|
| 62 张卡片数据量大，纯 JS 文件可能影响加载 | 按需加载，或静态 JSON + fetch |
| 移动端拖拽交互体验差 | 求助小游戏改为点击选择模式 |
| 用户替换图片后布局错乱 | CSS 使用固定宽高比容器 + `object-fit` |
| 每日重置依赖客户端时间，可被绕过 | 接受此限制，不做服务端校验（无后端项目） |

### 9.2 给用户的建议

1. **优先完成 Phase 0 的台词补充**——这是决定项目"灵魂"的核心资产。
2. **Phase 1 完成后即可内部体验**——不完美但完整，可以感受整体节奏。
3. **店员像素图不需要一次画完**——可先用 1 张占位，后续迭代。
4. **62 张卡片插图量大**——可先挑 10 张最核心的做，其余用 CSS 文字排版替代。
5. **音效是可选的锦上添花**——不要因为音效拖延上线。
