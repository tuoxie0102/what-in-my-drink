## 一、项目核心设定

### 1.1 一句话定义

一家由懒散店员经营的像素奶茶店。每杯奶茶杯底随机掉落一件别人遗失的旧物，附带一段真实存在过的普通人的记忆。

### 1.2 核心情绪

> "那个时候，那个人还在。"

用户不是来玩游戏。用户是走进一家有点奇怪的奶茶店，偶然捡到别人遗失的人生片段。

### 1.3 核心人物：店员

| 维度 | 设定 |
|------|------|
| 姓名 | 不愿透露，大家叫她"店员" |
| 性别 | 女性 |
| 年龄 | 看起来二十多岁 |
| 性格 | 懒散、摸鱼、经常不开店、不想上班但喜欢奶茶店 |
| 口头禅 | "今天开门纯属意外。" |
| 爱好 | 收集客人留下的物件，塞进奶茶里 |
| 气质 | 不是元气少女，不是丧系，是"想开店但不想上班" |

### 1.4 世界观设定

- 店员开店初期让客人用物件交换奶茶，后来柜子越来越满。
- 她开始把没人认领的东西"不小心"塞进奶茶里。
- 客人发现后不但不生气，反而传开了——越来越多人来找她买"带物件的奶茶"。
- 她不胜其扰，不想生意太好，经常挂"暂停营业"。

## 二、交互闭环

### 2.1 主循环

```
┌─────────────────────────────────────────────────┐
│                                                  │
│  ① 进入首页（像素奶茶店）                         │
│       ↓                                         │
│  ② 店员随机姿势 + 随机台词                        │
│       ↓                                         │
│  ③ 点击店员                                     │
│       ↓                                         │
│  ④ 今日菜单：三杯奶茶（2 普通 + 1 特惠）           │
│       ↓                                         │
│  ⑤ 选择一杯                                    │
│       ├── 普通奶茶 → 直接摇奶茶                   │
│       └── 特惠奶茶 → 店员二次确认（吐槽理由）→ 摇奶茶 │
│       ↓                                         │
│  ⑥ 摇奶茶动画                                  │
│       ↓                                         │
│  ⑦ 杯底掉落遗失物                               │
│       ↓                                         │
│  ⑧ 店员："……等一下。不好意思。这个也掉进去了。"     │
│       ↓                                         │
│  ⑨ 点击查看 → 记忆卡片（千禧年画报风）              │
│       ↓                                         │
│  ⑩ 店员吐槽（一句备注）                           │
│       ↓                                         │
│  ⑪ 物件进入收藏柜                                │
│       ↓                                         │
│  ⑫ 店员："今天有点累。明天再来拿吧。"              │
│       ↓                                         │
│  ⑬ 每日 3 杯喝完 → "今天卖完了" + 随机歇业理由     │
│                                                  │
└─────────────────────────────────────────────────┘
```

### 2.2 旁路触发：店员寻求帮助

随机触发（低概率），不占每日 3 杯额度：

| 帮助类型 | 交互方式 | 奖励 |
|----------|----------|------|
| 帮忙找吸管 | 从桌面杂物中点击吸管 | 店员顺手递一张车票，进入记忆卡片 |
| 帮忙封口 | 拖动封口膜到杯子上 | 封口膜下压着一张照片，进入记忆卡片 |
| 帮忙整理柜台 | 拖动物品归类 | 发现校卡背后有名字，进入记忆卡片 |

### 2.3 每日限制

- 每天 3 杯奶茶，喝完后显示"今天卖完了"。
- 随机显示歇业理由（高血糖、跳闸、封口机罢工、植脂末心理负担、营业额已超预期=12元等）。
- "第二天"基于用户本地日期重置。

## 三、页面架构

### 3.1 页面清单

| 页面 | 视觉风格 | 作用 |
|------|----------|------|
| **首页** | 像素奶茶店（星露谷/动森风） | 建立氛围，展示店员，入口 |
| **奶茶菜单** | 像素UI + 2000年代电脑桌面风 | 选择今日奶茶 |
| **店员确认弹窗** | 像素对话框 | 特惠奶茶二次确认 |
| **摇奶茶页** | 轻量动画（像素感） | 仪式感过渡 |
| **掉落物揭晓** | 奶茶杯裂开/倒出物件 | 悬念揭晓 |
| **记忆卡片页** | 千禧年画报风 | 情绪高潮，阅读故事 |
| **店员吐槽** | 像素对话框 | 收尾，幽默缓冲 |
| **收藏柜** | 像素展示柜 | 长期收集反馈 |
| **店员求助** | 像素桌面交互 | 随机小游戏 |
| **今日卖完** | 像素对话框 | 限制提示 |

### 3.2 页面转换视觉流

```
  像素奶茶店（首页）
       ↓
  像素UI + WinXP风（菜单）
       ↓
  轻量动画（摇奶茶）
       ↓
  ★ 千禧年画报风（记忆卡片）★ ← 情绪高潮
       ↓
  像素对话框（店员吐槽）
       ↓
  像素展示柜（收藏柜）
       ↓
  回到首页
```

**关键原则：** 像素风只存在于奶茶店和店员部分。遗失物/记忆卡片时切换为千禧年杂志、旧照片风格。

## 四、数据结构

### 4.1 奶茶（Drink）

```
{
  id: string,           // "school-pearl"
  name: string,         // "放学后的珍珠奶茶"
  category: string,     // "旧时代系列" | "城市系列" | "今日特调"
  era: string,          // "2006"
  price: number,        // 4
  period: string,       // "1998-2010"
  scene: string[],      // ["校园", "暑假"]
  is_special: boolean,  // 是否为特惠奶茶
  special_reason: string // 特惠原因（仅特惠奶茶）
}
```

共 **20 杯奶茶**：

| 类别 | 奶茶名称 |
|------|----------|
| 旧时代系列 | 放学后的珍珠奶茶、小卖部香芋奶茶、网吧冰红茶、优乐美同款奶茶、校门口双拼奶茶 |
| 城市系列 | 广州站冰红茶、城中村柠檬茶、CBD至尊冰城蜜雪、早古刨冰、兑水水果茶、随便加点酒精奶茶 |
| 今日特调 | 不想打工、租金又涨了、中个彩票吧、下班了明天再说、店员不良心推荐、喝一点吧、看个喜剧、还活着、高考后第一杯奶茶、暑假最后一天、毕业照那天、第一次约会、分手快乐、再见夏天 |

### 4.2 遗失物卡片（MemoryCard）

```
{
  id: string,              // "#001"
  drink_id: string,        // 所属奶茶
  name: string,            // "校服口袋里的合照"
  era: string,             // "2006年6月"
  location: string,        // "汕头某初中"
  background: string,      // 现实背景（时代现象）
  current_status: string,  // 现状（物是人非）
  item_description: string,// 物品描述（用户看到的物件）
  back_text: string,       // 背面内容（物件上的话）
  moment: string,          // 时刻描述（≤100字）
  clerk_note: string       // 店员备注（吐槽）
}
```

共 **62 张卡片**（#001–#062），详见 `奶茶与关联物品.md`。

### 4.3 店员台词库（ClerkLines）

```
{
  opening: string[],       // 首页随机台词（~10句）
  menu_intro: string[],    // 菜单介绍（~5句）
  special_confirm: string[],// 特惠确认理由（~8句）
  item_found: string[],    // 发现掉落物（~5句）
  after_story: string[],   // 讲故事后收尾（~8句）
  sold_out: string[],      // 卖完歇业理由（~10句）
  help_request: string[],  // 求助台词（~5句）
  help_thanks: string[],   // 帮助后感谢（~5句）
  idle: string[]           // 空闲随机吐槽（~15句）
}
```

**当前状态：** ✅ 已完成。详见 `店员台词库.md`（100 句，分 8 类）。

### 4.4 收藏柜（Collection）

```
{
  collected_items: string[], // 已收集的卡片 ID 列表
  last_visit_date: string,   // 最后访问日期（判断每日重置）
  daily_count: number        // 今日已喝杯数（0-3）
}
```

存储在 `localStorage`。

## 五、视觉风格系统

### 5.1 双轨风格

| 区域 | 风格 | 参考 |
|------|------|------|
| 奶茶店（首页） | 像素风 | 星露谷物语、动森、开罗游戏 |
| 奶茶菜单 | 像素UI + 2000年代桌面 | Windows XP、QQ2005、机房电脑 |
| 摇奶茶 | 轻量像素动画 | — |
| 记忆卡片 | 千禧年画报风 | 2000年代杂志、青春电影海报、QQ空间相册、数码相机照片 |
| 店员对话框 | 像素对话框 | RPG对话风格 |

### 5.2 色彩系统

| 用途 | 色系 |
|------|------|
| 奶茶店主色调 | 米白、奶茶色、浅棕、暖黄 |
| 禁止使用 | 霓虹灯、夜店感、高饱和赛博配色 |
| 记忆卡片 | 明亮、温暖、略微褪色、旧照片感 |

### 5.3 风格禁忌（明确禁止）

- 不做恐怖风
- 不做赛博朋克
- 不做蒸汽波
- 不做过度复古滤镜
- 不显示星级、掉率、SSR等游戏化数值
- 不制造抽卡感

### 5.4 彩蛋视觉风格

特调奶茶可触发视觉变化：

- 千禧年广告风
- 校园杂志风
- QQ空间风
- 数码相册风
- MP3播放器风

## 八、技术约束

1. **纯前端项目**：HTML + CSS + JavaScript，无后端。
2. **数据持久化**：`localStorage`，无数据库。
3. **图片可替换**：所有视觉资产路径集中在配置文件中，支持用户后期替换。
4. **不写死数据**：奶茶和卡片数据从 JS 数据文件加载，不硬编码在 HTML 中。
5. **响应式**：桌面优先，移动端需适配（摇奶茶改为点击触发）。
6. **零依赖优先**：尽量不引入框架，保持轻量。
